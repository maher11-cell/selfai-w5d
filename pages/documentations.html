Introduction

The W5D Kit is a static, build-step-free website system. Every file is a plain HTML page that loads Tailwind CSS via CDN, a tiny style.css for the design system, and a few small JS modules for interactivity (mega-menu, theme toggle, forms, animations).

It ships with three homepages, fourteen inner pages, twelve animated heroes, twenty-eight content sections, and four wired forms — enough to launch a complete marketing site in a day, or to reuse for any new product.

Stack: HTML + Tailwind (CDN) + vanilla JS + Lucide icons + a RESTful Table API. No bundler, no build step, no framework.

Quickstart

Want a new page that looks like the rest of the site? Copy this skeleton and fill the body.

1. Copy the page skeleton

<!DOCTYPE html>
<html lang="en" class="dark scroll-smooth">
<head>
<meta charset="UTF-8" /><meta name="viewport" content="width=device-width, initial-scale=1.0" />
<title>Your page — W5D</title>
<script src="https://cdn.tailwindcss.com"></script>
<script>tailwind.config={darkMode:'class',theme:{extend:{colors:{brand:{500:'#8b5cf6'},ink:{950:'#070711'}},fontFamily:{display:['"Space Grotesk"','Inter','sans-serif']}}}}</script>
<link rel="stylesheet" href="../css/style.css" />
<script src="https://unpkg.com/lucide@latest/dist/umd/lucide.js" defer></script>
<script src="../js/shell.js" defer></script>
<script src="../js/megamenu.js" defer></script>
<script src="../js/main.js" defer></script>
<script src="../js/api.js" defer></script>
</head>
<body data-prefix="../" class="bg-white text-slate-800 dark:bg-ink-950 dark:text-slate-200">
<div data-shell></div>
<!-- your sections here -->
<div data-shell-foot></div>
</body></html>
Set data-prefix="../" for files in /pages/. Use data-prefix="" (or omit it) for files in the project root.

2. Drop in a hero + sections

Open heros.html or sections.html, pick a block, and paste it between data-shell and data-shell-foot. The shell injects the navbar and footer automatically.

Brand & theme tokens

Tokens are defined in two places that you should always edit together: the inline tailwind.config in every page's <head>, and CSS variables in css/style.css.

Tailwind colors

brand.500#8b5cf6Primary CTAs, links, accents
brand.600#7c3aedGradient end, hover
ink.950#070711Page background (dark)
ink.900#0b0b18Surface (dark)
ink.800#11111fInset surface (dark)

CSS variables (gradients)

:root {
  --w5d-grad-1: #8b5cf6; /* primary */
  --w5d-grad-2: #6366f1; /* indigo  */
  --w5d-grad-3: #06b6d4; /* cyan    */
  --w5d-grad-4: #ec4899; /* pink    */
}

Fonts

Display: Space Grotesk (500/600/700). Body: Inter (300–800). Both are loaded from Google Fonts in every page. Use class="font-display" for headings.

Dark / light mode

The kit defaults to dark. The theme is stored in localStorage under w5d-theme and the html element gets the .dark class. main.js wires every [data-theme-toggle] button automatically.

<button data-theme-toggle aria-label="Toggle theme">
  <i data-theme-icon="sun"  data-lucide="sun"  class="w-5 h-5"></i>
  <i data-theme-icon="moon" data-lucide="moon" class="w-5 h-5 hidden"></i>
</button>

For dark-aware styling use Tailwind's dark: variant — e.g. bg-white dark:bg-ink-950, text-slate-800 dark:text-slate-200.

Page shell (auto nav + footer)

Every inner page gets its navbar and footer injected by js/shell.js. You only need two markers:

<body data-prefix="../">
  <div data-shell></div>        <!-- nav goes here -->
  <!-- your page content -->
  <div data-shell-foot></div>   <!-- footer goes here -->
</body>

The data-prefix on <body> tells the shell where the project root is. Set it to "" on root pages and "../" on pages in /pages/.

Mega-menu

The mega-menu is data-driven — edit the NAV object at the top of js/megamenu.js to add or remove items. Each top-level key becomes a hover trigger; each group becomes a column; each entry becomes a styled link with icon + description.

Add a new menu item

// js/megamenu.js
product: {
  title: 'Product',
  groups: [
    { heading: 'Platform', items: [
      { i: 'sparkles',  t: 'Features overview', d: 'Everything in one map.', href: 'pages/features.html' },
      { i: 'rocket',    t: 'Your new page',     d: 'Short description.',     href: 'pages/your-page.html' }
    ]}
  ],
  promo: { tag: 'New', title: 'Featured story', body: 'Pitch line', href: 'pages/blog-post.html' }
}

Icon names use the Lucide icon set.

The panel uses an internal padding bridge plus a 120ms close-delay so the cursor can travel from trigger to panel without the menu disappearing — a common UX bug fixed once for all pages.

Reveal on scroll

Add data-rv to any element. It fades in when scrolled into view. Pair with .stagger on the parent to animate children one after the other.

<div data-rv="up" class="stagger">
  <h2>Headline</h2>
  <p>Subtitle</p>
  <a>CTA</a>
</div>

<!-- variants -->
data-rv="up" | "down" | "left" | "right" | "zoom" | "rotate"

Forms & Table API

Drop a single attribute on a <form> and it's wired to the right RESTful endpoint with success/error states. js/api.js handles everything.

SelectorEndpointRequired fields
form[data-newsletter]POST /tables/subscribersemail
form[data-contact]POST /tables/contact_messagesname, email, message
form[data-demo]POST /tables/demo_requestsname, work_email, company
form[data-careers]POST /tables/careers_applicationsname, email, role

Example: contact form

<form data-contact class="space-y-4">
  <input name="name"     required type="text"  placeholder="Full name" />
  <input name="email"    required type="email" placeholder="Email" />
  <input name="company"           type="text"  placeholder="Company" />
  <select name="topic">
    <option value="sales">Sales</option>
    <option value="support">Support</option>
  </select>
  <textarea name="message" required rows="5"></textarea>
  <button type="submit">Send message</button>
</form>
Field name attributes must match the table schema exactly. js/api.js reads them via FormData.

Database schemas

Four tables back the wired forms. Each row gets a UUID id and timestamps automatically.

TableFields
subscribersemail, source_page, status

REST endpoints

  • GET /tables/{table}?page=1&limit=100
  • GET /tables/{table}/{id}
  • POST /tables/{table}
  • PATCH /tables/{table}/{id}
  • DELETE /tables/{table}/{id}

Accessibility

  • Focus rings: visible 2px brand-color rings on all focusable elements (:focus-visible).
  • Mega-menu: aria-haspopup, aria-expanded, focus-within open, Escape closes.
  • Mobile menu auto-closes when a link is tapped.
  • All animations honour prefers-reduced-motion.
  • Use semantic landmarks (header, main, section, footer) — never wrap with bare div when a tag fits.

Performance

  • All scripts use defer — HTML parses first, no blocking.
  • Animations are GPU-friendly (transform / opacity only).
  • No images by default — all visuals are CSS / SVG / icon font.
  • Lucide icons load once on the DOMContentLoaded tick.
  • The Tailwind CDN warning is expected. For production, compile Tailwind via the CLI to drop the bundle to ~20 KB.

Deploy

The site is fully static — drop the project folder on any static host (the Publish tab in this environment, or Netlify, Vercel, S3, GitHub Pages, etc.). The Table API endpoints are relative, so they'll route through whatever host you publish on.

To deploy in this environment, open the Publish tab — it handles everything in one click.

Rebranding playbook

Reusing the kit for a new brand takes about an hour:

  1. Find/replace W5D with your new name (across all HTML + JS).
  2. Update the logo: edit the W5 mark inside js/shell.js and the three home pages.
  3. Swap brand colors: change brand.500 in every tailwind.config block and the --w5d-grad-* variables in css/style.css.
  4. Replace the fonts in the Google Fonts <link> if you want a different display family.
  5. Update the NAV object in js/megamenu.js to match your IA.
  6. Rewrite copy on the three homepages — everything else cascades.
Tip: most editors can do a project-wide find/replace. Be sure to include js/ as well as *.html.

FAQ

Why no build step?
Because most marketing sites don't need one. CDN Tailwind is fine for prototypes and small sites. Compile when you outgrow it.
Can I add a router / SPA?
You can, but the kit is intentionally MPA-first — every page is a real HTML file. Easier SEO, easier sharing, easier debugging.
How do I add a new form?
Define a new table via TableSchemaUpdate, then either use one of the existing data-* selectors or extend js/api.js with a new handler.
Where do animations come from?
All custom animations are in css/style.css as utility classes. AOS is loaded as a fallback on some pages but the in-house data-rv system is preferred.
